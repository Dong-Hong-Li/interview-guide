# 文档索引（`interview-guide-go-take-notes/docs`）

> 本目录是仓库唯一的文档入口；所有架构、规范、流程描述以本目录为准，**不**抄主项目 Java 后端 `modules/*` 或 `router/router.go` 的目录习惯。

## 文档清单

| 文档 | 适用读者 | 主要回答 |
|------|----------|----------|
| [`项目架构.md`](./项目架构.md) | 新人、跨域开发者、SRE | 仓库长什么样、各层职责、HTTP/异步链路、依赖装配、状态机 |
| [`开发规范.md`](./开发规范.md) | 开发与 Reviewer | 包/命名/HTTP 绑定/错误日志/Wire/SQL/测试/自检清单 |
| [`项目管理.md`](./项目管理.md) | 全员（含 PM/SRE） | 分支、提交、PR、CI、版本、环境、部署、回滚、Issue |
| [`开发进度.md`](./开发进度.md) | 跨项目对账 | 当前已实现 / 占位 / 与主项目 Go 能力差异 |

## 阅读顺序建议

1. 第一次接触仓库 → `项目架构.md` §1~§5（顶层目录、分层、入口、HTTP 壳）；
2. 准备写代码 → `开发规范.md` §2~§9（包结构、HTTP、服务、端口、错误日志）；
3. 准备提 PR → `开发规范.md` §16（自检清单）+ `项目管理.md` §3~§5（提交、PR、CI）；
4. 准备发布 → `项目管理.md` §6~§9（版本、部署、监控、回滚）；
5. 接口对账 → `开发进度.md`（端点表 + 占位列表）。

## 维护守则

- 改了路由 / 端口 / 状态机 / 配置 → **同一 PR** 内同步以上文档；
- `开发进度.md` 第 6 行的「文档同步日期」必须更新为合并日；
- 删除/重命名文档需在本 README 同步更新链接；
- 新增文档放 `docs/<topic>.md`，并在「文档清单」表追加一行；如属于设计稿放 `docs/design/<topic>.md`（参见 `项目管理.md` §14）。

## 与主项目的关系

- 业务语义（接口形状、状态机、错误文案）以**主项目 Java 后端**为对齐基准；
- 实现细节（DI、Mapper 即 Adapter、消费者、PDF）以**本仓库代码**为准；
- 冲突时优先级：**代码 > `开发进度.md` > `项目架构.md` > 主项目 Java 文档**。
